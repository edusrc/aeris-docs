# Directives

Assembler directives are special commands that control how the assembler organizes
data and instructions in memory. Unlike normal instructions, directives **do not
generate machine code**. Instead, they affect how the program is assembled.

ÆRIS currently supports three main data directives:

- `.word`
- `.ascii`
- `.string`

These directives are used inside the **`.data` section** to define values that will
be stored in memory before program execution begins.

---

# `.word`

The `.word` directive stores **32-bit integers** in memory.

Each value occupies **4 bytes** and is stored using **little-endian byte order**.

Example:

```asm
.data
value: .word 10
```

Memory layout:

```
address       value
0x10010000    0x0000000A
```

Multiple values can be defined in one directive:

```asm
.data
numbers: .word 1, 2, 3, 4
```

This produces four consecutive 32‑bit words in memory.

---

# `.ascii`

The `.ascii` directive stores a **sequence of characters** in memory.

Unlike `.string`, it **does not automatically append a null terminator (`\0`)**.

Example:

```asm
.data
msg: .ascii "Hello"
```

Memory layout:

```
48 65 6C 6C 6F
```

This corresponds to the ASCII characters:

```
H e l l o
```

Because there is no null terminator, `.ascii` is mainly useful when storing raw
character sequences or binary data.

---

# `.string`

The `.string` directive stores a character sequence **with a null terminator**.

The assembler automatically appends a `\0` byte at the end of the string.

Example:

```asm
.data
msg: .string "Hello"
```

Memory layout:

```
48 65 6C 6C 6F 00
```

This format is commonly used for strings that will be printed using syscalls such
as `PrintString`.

Example usage with a syscall:

```asm
.data
msg: .string "Hello RISC-V\n"

.text
main:
    li a7, 4
    la a0, msg
    ecall
```

---

# Memory Alignment

Data generated by directives is aligned to **4-byte word boundaries** when required.
If a directive produces a number of bytes that is not a multiple of 4, padding
may be inserted so that the next data item begins at a word-aligned address.

This simplifies memory access for instructions such as `lw` and `sw`.

---

# Example Data Section

Example combining all directives:

```asm
.data

number:  .word 42
text1:   .ascii "ABC"
text2:   .string "Hello"
```

Resulting memory layout:

```
address       data
--------------------------------
0x10010000    2A 00 00 00
0x10010004    41 42 43
0x10010007    48 65 6C 6C 6F 00
```

The assembler ensures the next symbol begins at the correct aligned address.

---

# Summary

| Directive | Purpose |
|----------|--------|
| `.word` | Store 32‑bit integer values |
| `.ascii` | Store characters without null terminator |
| `.string` | Store characters with null terminator |

These directives allow programs to define data that can later be accessed
through normal load and store instructions.
